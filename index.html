📋 自动化项目总览

以下所有系统均在 PatPat.com 电商业务中实际运行，使用 AI 编程工具完成。

🗂️ 1. G+ 拆单自动化

Node.js Playwright ⏱ 定时运行
❓ 之前：G+ 异常订单需要人工判断是否拆单、手动操作拆单流程，耗时且易出错。
✅ 之后：自动从数据库拉取异常订单 → 库存匹配运算 → 生成拆单方案 → 自动打开浏览器执行拆单 → 结果回填飞书表格。
G+ 平台 MySQL 飞书表格 Playwright

📦 2. 缺货订单自动化排查

Python ⏱ 每天 09:00 / 14:00 / 17:00
❓ 之前：客服每天人工查两个系统的缺货订单，整理 Excel、手动发邮件、容易漏单。
✅ 之后：自动从 G+ 异常订单和 WMS 待发货订单识别真正缺货的订单（区分中美库存逻辑），同步飞书，自动高亮优先级，定时邮件通知。
G+ 平台 WMS 仓储 飞书多维表格 邮件

💬 3. CS Task 缺货自动跟进

Python ⏱ 每 90 分钟
❓ 之前：客服工单中的缺货问题需要人工分类、查库存、写邮件回复客户，流程繁琐。
✅ 之后：自动分类识别缺货工单 → 查 Shopify 客户信息 → 自动生成并发送 Intercom 邮件 → 24h 检测回复 → 未回复二次跟进或转人工。每小时刷新存量状态。
飞书 Base Shopify Intercom 值班表

✉️ 4. 异常订单 AI 邮件自动跟进

Python AI 生成 ⏱ 每小时
❓ 之前：异常订单需要客服手动编写、审核、发送邮件，漏跟延跟频繁。
✅ 之后：飞书 Base 读取异常订单 → 自动分流 → Intercom 搜索客服记录 → AI 自动生成邮件草稿 → 审核后自动发送 → 24h 检测回复状态 → 未回复二次跟进。
飞书 Base Intercom Shopify AI 模型

📱 5. Meta 评论 AI 回复自动化

Python Meta Graph API AI 分类
❓ 之前：Facebook/Instagram 客户评论人工一条条看、一条条回复，效率极低。
✅ 之后：通过 Meta Graph API 自动拉取评论 → AI 理解并分类（咨询/投诉/好评等） → 生成建议回复 → 写入飞书 → 人工确认后发出。
Meta Graph API 飞书多维表格 AI 模型 Webhook

🏷️ 6. 标发失败检测（Mark Ship Alert）

Python ⏱ 每天 09:00 / 17:00
❓ 之前：标发失败的订单需要人工逐个排查，跨系统比对，容易遗漏。
✅ 之后：自动查询 G+ 数据库标发失败订单 → 交叉比对 Shopify 发货状态 → 过滤真正需要处理的订单 → 发送预警邮件。支持 PatPat + Mamazing 双店铺。支持健康检查、日志查询。
G+ 数据库 Shopify 邮件

📊 7. Shopify 退款率周报自动化

Python ⏱ 每周
❓ 之前：每周需要手工拉取 Shopify 订单数据做退款率分析，耗时且口径不统一。
✅ 之后：自动按退款发生周统计退款金额率，两周对比分析，生成完整 Excel 报表（含退款原因分类、取消原因分类、事件明细），支持权限不足时自动降级。
Shopify GraphQL API Excel 报表

Python launchd ⏱ 每 5 分钟 / 每天 18:30
